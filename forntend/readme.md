# 前端技术架构（MVP）

## 1. 核心框架

- **Next.js 14+**（App Router）+ **TypeScript**
  - 服务端渲染（SSR）提升首屏性能
  - 文件路由系统，开发体验好
  - TypeScript 提供类型安全

## 2. UI 框架

- **Tailwind CSS**：原子化 CSS，快速构建界面
- **Ant Design**：企业级 UI 组件库
  - 提供输入框、按钮、卡片等基础组件
  - 适合快速开发

## 3. 其他工具

- **TanStack Query (React Query)**：服务端状态管理
  - 自动缓存、重试、后台更新
  - 适合 API 数据获取
- **Zustand**：客户端状态管理（轻量级）
  - 全局状态（用户信息、UI 状态等）
  - 比 Redux 更简单，适合 MVP

## 4. MVP 核心页面（最小功能）

### 检索页（唯一页面）

**功能清单**
- ✅ 问题输入框（支持多行文本输入）
- ✅ 检索按钮（触发搜索）
- ✅ 返回数据展示框（显示检索结果）

**页面布局**
```
┌─────────────────────────────────────┐
│  问题输入框（多行文本）              │
│  ┌───────────────────────────────┐  │
│  │                               │  │
│  │  请输入要检索的问题...        │  │
│  │                               │  │
│  └───────────────────────────────┘  │
│  [检索] 按钮                        │
├─────────────────────────────────────┤
│  返回数据展示框                     │
│  ┌───────────────────────────────┐  │
│  │  检索结果：                   │  │
│  │  - 文档1：xxx                 │  │
│  │  - 文档2：xxx                 │  │
│  │  ...                          │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

**数据展示内容**
- 文档名称
- 文档版本
- 需求编号（如有）
- 匹配的文本片段
- 相似度评分

## 5. MVP 开发优先级

### Phase 1（第1周）
1. ✅ 项目初始化（Next.js + TypeScript + Tailwind + Ant Design）
2. ✅ 检索页基础布局（输入框 + 按钮 + 结果展示框）
3. ✅ API 客户端封装（TanStack Query）
4. ✅ 检索功能对接后端

### Phase 2（第2周）
1. ✅ 结果数据格式化展示
2. ✅ 加载状态提示
3. ✅ 错误处理
4. ✅ 基础样式优化

### Phase 3（后续扩展）
- 关键词高亮
- 引用预览卡片
- 跳转到飞书文档
- 项目/文档筛选
- 其他高级功能
